### 接到新任務時：
1. 在 REF_status. md 將任務狀態改為 “進行中”
2. 確認 REF_prd. md 中的驗收標準
3. 檢查 REF_techspec. md 相關技術規範
4. 建立 feature branch: git checkout -b feature/task-name

### 程式碼品質
- [ ] 程式碼符合命名規範
- [ ] 沒有硬編碼的魔術數字或字串
- [ ] 函數長度合理（建議 < 50 行）
- [ ] 複雜邏輯有適當註解說明
- [ ] 沒有重複程式碼（DRY 原則）
- [ ] 變數和函數名稱語意清楚
### 功能完整性
- [ ] 符合 REF_prd. md 中的需求規格
- [ ] 所有邊界條件都有處理（空音檔、超長音檔等）
- [ ] 錯誤處理完整且用戶友善
- [ ] 音頻輸入驗證充足
- [ ] 轉錄結果符合預期格式（繁體中文 + 標點符號）
### 效能與安全
- [ ] 沒有明顯的效能瓶頸（轉錄速度符合要求）
- [ ] 記憶體使用量在合理範圍內
- [ ] 本地資料有適當保護（暫存檔案清理）
- [ ] 音頻檔案處理安全（格式驗證）
- [ ] 無資料外洩風險（確保本地處理）
